# Food Codifier Demo

Демонстрационный веб-прототип для автоматической кодификации записей рациона питания по статистическому кодификатору.

Специалист видит записи рациона, запускает автоматическое сопоставление и может:

- принять предложенный код;
- выбрать альтернативного кандидата;
- отправить запись на ручную проверку;
- сбросить принятое решение;
- обработать все записи одной командой.

> Это учебный демонстрационный проект. Алгоритм использует текстовое сходство и эвристики, а не обученную ML/LLM-модель. Результаты нельзя применять как официальный статистический классификатор без валидации на реальных данных.

## Что находится в проекте

- `data/ration_week.json` — 49 демонстрационных записей рациона;
- `data/demo_food_codifier_1000.json` — 1000 демонстрационных позиций кодификатора;
- `artifacts/api-server` — API на Express;
- `artifacts/food-codifier` — интерфейс на React + Vite;
- `artifacts/api-server/src/lib/matcher.ts` — алгоритм сопоставления;
- `lib/api-spec/openapi.yaml` — контракт API.

## Локальный запуск на Windows

### 1. Установите необходимые программы

- Node.js версии 20 или новее;
- Git — для работы с GitHub;
- Visual Studio Code — необязательно, но удобно.

Проверьте Node.js в PowerShell:

```powershell
node -v
npm -v
```

### 2. Включите pnpm

```powershell
corepack enable
corepack prepare pnpm@10.15.0 --activate
pnpm -v
```

Если Windows не разрешает `corepack enable`, можно установить pnpm через npm:

```powershell
npm install -g pnpm@10.15.0
```

### 3. Установите зависимости

Откройте PowerShell в корне проекта и выполните:

```powershell
pnpm install
```

### 4. Запустите приложение

```powershell
pnpm dev
```

После запуска откройте:

```text
http://localhost:5173
```

API работает по адресу:

```text
http://localhost:8080/api/healthz
```

Для остановки нажмите `Ctrl + C` в терминале.

## Важное отличие от исходной версии Replit

Версия в этом репозитории подготовлена для локального запуска:

- не требует Replit;
- не требует API-ключей;
- не требует PostgreSQL;
- frontend перенаправляет запросы `/api` на локальный API-сервер;
- решения специалиста сохраняются локально в `data/codification_results.local.json`.

Файл с локальными решениями не попадает в GitHub и создаётся автоматически при первом действии пользователя.

## Команды

```powershell
pnpm dev            # frontend и API одновременно
pnpm dev:web        # только frontend
pnpm dev:api        # только API
pnpm validate:data  # проверка структуры JSON-файлов
pnpm test           # проверка данных и smoke-test алгоритма
pnpm typecheck      # проверка TypeScript
pnpm build          # production-сборка
```

## Настройка портов

Стандартные значения уже заданы. Для изменения скопируйте `.env.example` в `.env` и отредактируйте:

```env
API_PORT=8080
WEB_PORT=5173

```

## Как работает сопоставление

Для каждой записи алгоритм сравнивает описание блюда с позициями кодификатора по нескольким признакам:

- название;
- синонимы;
- ключевые слова;
- описание;
- раздел и подгруппа;
- способ приготовления;
- единица измерения.

Система возвращает пять наиболее близких кандидатов и рассчитывает демонстрационную оценку уверенности.

## Ограничения текущего прототипа

- демоданные не являются официальным кодификатором;
- качество сопоставления не оценивалось на размеченной выборке;
- значение confidence является эвристикой, а не статистически откалиброванной вероятностью;
- нет авторизации и разделения пользователей;
- локальное хранилище рассчитано на одного пользователя и демонстрационный сценарий.
